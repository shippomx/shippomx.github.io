Go语言常见内存泄露

哪吒藕霸2020-02-232023-09-28

Go语言常见内存泄露

1、没有正确的关闭或未关闭连接。包括数据库连接，队列连接，第三方库，文件…
2、没有正确使用defer()函数
3、使用大字符串/切片/子序列对其做多次操作。
4、指针指向不使用的值
5、悬挂Goroutine
6、误用Finalizer

检查Go程序内存的使用情况最常用的就是Go标准库自带的pprof库了，可以通过http暴露出这个profile, 然后通过go tool pprof或者pprof工具命令行/web方式查看。

比如下面的命令， 可以获取服务器http://ip:port的堆信息，并且在本机9090端口启动一个服务器展示堆的信息。

在堆信息中你可以查看分配的堆的大小和对象数量，或者当前没有释放的占用的堆的大小和对象数量。

正常情况下使用这个方式就可以比较直观的看到哪一段代码分配的内存比较多，然后确定那里容易产生内存泄漏。

但是, 分配堆内存比较多的地方并不一定产生内存泄漏，只能说明这个地方”曾经/正在”分配的堆内存比较大,或者分配的堆内存比较频繁俄安，这些分配的内存可能在之后就回收掉了。

pprof也可以比较两个时间点的分配的内存的差值，通过比较差值，就容易看到哪些地方产生的内存”残留”的比较多，没有被内存释放，极有可能是内存泄漏的点。

可以通过下面的方式产生两个时间点的堆的profile,之后使用pprof工具进行分析。

  1. 首先确保你已经配置了pprof的http路径， 可以访问http://ip:port/debug/pprof/查看(如果你没有修改默认的pprof路径)
  2. 导出时间点1的堆的profile: curl -s http://127.0.0.1:8080/debug/pprof/heap > base.heap, 我们把它作为基准点
  3. 等待一段时间后导出时间点2的堆的profile: curl -s http://127.0.0.1:8080/debug/pprof/heap > current.heap
  4. 现在你就可以比较这两个时间点的堆的差异了: go tool pprof --base base.heap current.heap

直接使用命令打开web界面: go tool pprof --http :9090 --base base.heap current.heap

  • topN 命令, 默认能列出当前程序中内存占用排名前 10 的函数. 每一行表示一个函数的信息。前两列表示函数在 CPU 上运行的时间以及百分比；第三列是当前所有函数累加使用 CPU 的比例；第四列和第五列代表这个函数以及子函数运行所占用的时间和比例（也被称为累加值 cumulative），应该大于等于前两列的值；最后一列就是函数的名字。

  • list <函数名>, 展现函数内部的内存占用. 前两列表示函数运行时占用的内存以及百分比；第三列是当前所有函数累加使用内存 的比例；第四列和第五列代表这个函数以及子函数运行所占用的内存和比例（也被称为累加值 cumulative），应该大于等于前两列的值；最后一列就是函数的名字。
