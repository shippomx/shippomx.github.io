Go package的组织方式

应该要做到这几点:

  1. 依赖管理
  2. 依赖包版本控制
  3. 对应的包管理平台
  4. 可以私有化部署

加分：

  1. 代码包是否可以复用
  2. 构建，测试,打包
  3. 发布上线

comman description
go get golang.org/x/text@latest 拉取最新的版本(优先择取 tag)

go get golang.org/x/text@342b2e 拉取 hash 为 342b231 的 commit，最终会被转换为 v0.3.2

go get -u 更新

command description

3.2.3 go mod edit

command description
go mod edit -fmt 格式化go.mod文件
go mod edit -module 指定go.mod路径
go mod edit -require=path@version 在给定的模块路径和版本上添加一个依赖
go mod edit -droprequire=path 在给定的模块路径删除一个依赖
go mod edit -replace=old[@v]=new[@v] 替换下载不了的路径, 见上面说明
go mod edit -dropreplace=old[@v] 废弃之前的替换
go mod edit -exclude=path@version 排除路径
go mod edit -dropexclude=path@version 废弃之前的exclude
go mod edit -retract=version 指定go.mod路径
go mod edit -dropretract=version 指定go.mod路径
go mod edit -go=version 指定go版本
go list -m -json all 以json格式查看项目整个依赖
